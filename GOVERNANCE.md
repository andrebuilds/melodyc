# Governance

## Purpose

This document describes how Melodyc is maintained and how project decisions are made.

## Maintainers

Melodyc is maintained by the project authors listed in [MAINTAINERS.md](MAINTAINERS.md). Maintainers are responsible for reviewing contributions, managing releases, enforcing the [Code of Conduct](CODE_OF_CONDUCT.md), and protecting the project's long-term health.

## Decision Making

Maintainers make day-to-day decisions, including bug fixes, documentation changes, dependency updates, and pull request approvals.

For significant changes, such as new major features, architectural changes, changes to public APIs, or changes that affect the project's scope, contributors should open an issue before implementation. Maintainers will consider technical feasibility, maintenance cost, compatibility, security, and community feedback.

The maintainers aim to reach consensus. When consensus cannot be reached, the lead maintainer makes the final decision.

## Becoming a Maintainer

New maintainers may be invited based on sustained, constructive contributions to the project, including code, documentation, issue triage, reviews, and community support.

Potential maintainers must demonstrate:

- Familiarity with the project and its technical direction
- Respectful and reliable collaboration
- Consistent adherence to the [Code of Conduct](CODE_OF_CONDUCT.md)
- Willingness to help maintain the project over time

## Changes to This Document

Changes to this governance model should be proposed through an issue or pull request. Significant changes require agreement from the current maintainers.
