## 1. Prerequisites

Before starting, make sure you have installed:

- **Python 3.12** — [python.org/downloads](https://www.python.org/downloads/)
- **pip** (included with Python)
- A **GitHub account** to clone the repo

Verify the Python installation:

```bash
python --version
# Expected output: Python 3.12.x
```

---

## 2. Create a Modal account

Modal is the serverless platform that runs Python code on GPU in the cloud. Every time a user generates a song, Modal starts a GPU container and runs the AI models.

### Steps

1. Go to [modal.com](https://modal.com) and click **Sign Up**
2. Sign up with GitHub (recommended) or with email
3. Complete the onboarding: enter the workspace name (e.g. `melodyc`)
4. Modal offers **$30 in free credits per month** — more than enough for development and testing

> **Note:** The free plan includes the L40S GPU which is the one used in this project. A credit card is not required to get started.

---

## 3. Set up Modal locally

### 3.1 Install the Modal CLI

After activating your Python environment (see section 8), install Modal:

```bash
pip install modal
```

### 3.2 Authenticate

Run the setup command that opens the browser and authenticates the CLI with your account:

```bash
modal setup
```

Follow the instructions in the browser. At the end you will see in the terminal:

```
Web authentication finished successfully!
```

### 3.3 Verify the connection

```bash
modal profile current
```

You should see the name of your Modal workspace.

### 3.4 Core Modal concepts

| Concept | Description |
|---|---|
| `modal.App` | The Modal application (corresponds to a project). In Melodyc it's called `"melodyc"` |
| `modal.Image` | The Docker image with installed dependencies |
| `@app.cls(gpu="L40S")` | Class executed on L40S GPU in the cloud |
| `@modal.enter()` | Method executed once at container startup (loads models) |
| `@modal.fastapi_endpoint()` | Exposes a method as a REST HTTP endpoint |
| `modal.Volume` | Persistent storage between executions (used to cache AI models) |
| `modal.Secret` | Encrypted environment variables (AWS credentials, etc.) |

## 4. Create an AWS account

AWS is used to store the generated audio files (`.wav`) and cover art (`.png`) via the **S3** (Simple Storage Service) service.

### Steps

1. Go to [aws.amazon.com](https://aws.amazon.com) and click **Create an AWS Account**
2. Enter email, password and account name (e.g. `melodyc`)
3. Choose the **Free Tier** plan (free)
4. Enter credit card details (required by AWS, but you won't be charged for basic S3 usage)
5. Complete the phone number verification
6. Select the **Basic Support** plan (free)
7. Access the **AWS Console** at [console.aws.amazon.com](https://console.aws.amazon.com)

> **Recommended region:** Select `us-east-1` (N. Virginia) as the default region in the top right of the console — it's the most affordable and has the lowest latency with Modal.

---

## 5. Create an S3 bucket

The S3 bucket is the "container" where all files generated by Melodyc are saved.

### 5.1 Open S3

1. In the AWS Console search for **S3** in the search bar
2. Click on **Create bucket**

### 5.2 Configure the bucket

| Field | Value |
|---|---|
| Bucket name | `melodyc-bucket` (must be globally unique) |
| AWS Region | `us-east-1` |
| Object Ownership | ACLs disabled (default) |
| Block Public Access | **Keep everything blocked** ✅ |
| Versioning | Disabled |
| Encryption | SSE-S3 (default) |

3. Click **Create bucket**

> **Important:** Never make the bucket public. Access to files is done via **presigned URLs** generated server-side, which expire after 1 hour.

### 5.3 Note the bucket name

Save the bucket name (e.g. `melodyc-bucket`) — you'll need it as an environment variable.

## 6. Create IAM users

IAM (Identity and Access Management) allows you to create credentials with limited permissions. You need **two**: one for the backend (Modal) and one for the frontend (Next.js).

### 6.1 Open IAM

1. In the AWS Console search for **IAM**
2. In the left menu click **Users** → **Create user**

---

### 6.2 Backend User (Modal)

This user has permission to **upload and read** files on S3 — it is used by the Python backend to upload audio and images.

**Step 1 — User details:**
- User name: `melodyc-backend`
- Click **Next**

**Step 2 — Permissions:**
- Select **Attach policies directly**
- Click **Create policy** (opens a new tab)

In the new tab, select **JSON** and paste this policy:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowUploadAndRead",
            "Effect": "Allow",
            "Action": [
                "s3:PutObject",
                "s3:GetObject"
            ],
            "Resource": "arn:aws:s3:::melodyc-bucket/*"
        },
        {
            "Sid": "AllowList",
            "Effect": "Allow",
            "Action": "s3:ListBucket",
            "Resource": "arn:aws:s3:::melodyc-bucket"
        }
    ]
}
```

- Policy name: `MelodycBackendPolicy`
- Click **Create policy** and return to the previous tab
- Refresh the list and select `MelodycBackendPolicy`
- Click **Next** → **Create user**

**Step 3 — Create Access Keys:**
1. Click on the `melodyc-backend` user just created
2. Go to the **Security credentials** tab
3. Scroll to **Access keys** → click **Create access key**
4. Select **Application running outside AWS**
5. Click **Next** → **Create access key**
6. **⚠️ IMPORTANT:** Save `Access key ID` and `Secret access key` immediately — you won't be able to see them again

---

### 6.3 Frontend User (Next.js)

This user has permission to **read** files and generate presigned URLs — it is used by the frontend to let users listen to tracks.

**Step 1 — User details:**
- User name: `melodyc-frontend`
- Click **Next**

**Step 2 — Permissions:**
- Click **Create policy** and paste:

```json
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "AllowRead",
            "Effect": "Allow",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::melodyc-bucket/*"
        },
        {
            "Sid": "AllowList",
            "Effect": "Allow",
            "Action": "s3:ListBucket",
            "Resource": "arn:aws:s3:::melodyc-bucket"
        }
    ]
}
```

- Policy name: `MelodycFrontendPolicy`
- Return to the user, select the policy and complete the creation

**Step 3 — Create Access Keys** (same procedure as backend):
- Save `Access key ID` and `Secret access key` for `melodyc-frontend`

## 7. Configure Secrets on Modal

Modal Secrets are encrypted environment variables that the backend can read securely at runtime. In `main.py` they are loaded with:

```python
melodyc_secrets = modal.Secret.from_name("melodyc-secret")
```

### 7.1 Create the Secret

1. Go to [modal.com](https://modal.com) → access your workspace
2. In the left menu click **Secrets** → **Create secret**
3. Choose **Custom** as the type
4. Secret name: `melodyc-secret`
5. Add the following environment variables:

| Key | Value |
|---|---|
| `AWS_ACCESS_KEY_ID` | Access key ID of the `melodyc-backend` user |
| `AWS_SECRET_ACCESS_KEY` | Secret access key of the `melodyc-backend` user |
| `AWS_DEFAULT_REGION` | `us-east-1` |
| `S3_BUCKET_NAME` | `melodyc-bucket` (or the name you chose) |

6. Click **Create**

> The backend reads `S3_BUCKET_NAME` with `os.environ["S3_BUCKET_NAME"]` and the AWS credentials are automatically read by `boto3` through standard AWS variables.

### 8.1 Navigate to the backend folder

## 9. Code structure

```
backend/
├── main.py          # Main entry point — Modal App, API endpoints
├── prompts.py       # Prompt templates for Qwen2 (text and category generation)
├── requirements.txt # Python dependencies
└── ACE-Step/        # ACE-Step model submodule (do not modify)
```

### 9.1 main.py — Overview

```python
app = modal.App("melodyc")          # App name on Modal

image = (                           # Docker image with all dependencies
    modal.Image.debian_slim()
    .pip_install_from_requirements("requirements.txt")
    .run_commands(["git clone https://github.com/ace-step/ACE-Step.git ..."])
)

model_volume = modal.Volume.from_name("ace-step-models", ...)  # Model cache
hf_volume = modal.Volume.from_name("qwen-hf-cache", ...)       # HuggingFace cache
```

The `MusicGenServer` class is the core of the backend:

- `@modal.enter()` → `load_model()`: loads the 3 AI models at container startup
- `generate_from_description()`: receives a free description and generates prompt + lyrics with Qwen2, then generates audio with ACE-Step
- `generate_with_lyrics()`: receives prompt + already written lyrics, generates only the audio
- `generate_with_described_lyrics()`: receives prompt + lyrics description, generates lyrics with Qwen2 then audio

### 9.2 prompts.py — Templates

Contains the templates for the prompts sent to the Qwen2 model:
- `PROMPT_GENERATOR_PROMPT` — transforms a description into audio tags (genre, instruments, BPM, etc.)
- `LYRICS_GENERATOR_PROMPT` — generates structured lyrics with `[verse]`, `[chorus]`, `[bridge]`, etc.

## 10. Run and deploy the backend

### 10.1 Run locally (test)

This command runs the `local_entrypoint` defined in `main.py` — useful for testing the full flow before deploying:

```bash
modal run main.py
```

On first run Modal:
1. Builds the Docker image (may take a few minutes)
2. Loads the AI models (ACE-Step, Qwen2, SDXL-Turbo) into the volumes — only the first time, then they are cached
3. Runs the `main()` function of the entrypoint

### 10.2 Deploy the backend

The deploy makes the HTTP endpoints permanently available (they don't shut down after the test):

```bash
modal deploy main.py
```

After the deploy you will see in the terminal the endpoint URLs, similar to:

```
✓ Created web endpoint melodyc-musicgenserver-generate-from-description
  https://andrebuils--melodyc-musicgenserver-generate-from-description.modal.run
✓ Created web endpoint melodyc-musicgenserver-generate-with-lyrics
  https://andrebuils--melodyc-musicgenserver-generate-with-lyrics.modal.run
✓ Created web endpoint melodyc-musicgenserver-generate-with-described-lyrics
  https://andrebuils--melodyc-musicgenserver-generate-with-described-lyrics.modal.run
```

**Copy these URLs** — they will need to be entered as environment variables in the frontend.

### 10.3 Monitor executions

Go to [modal.com](https://modal.com) → **Apps** → `melodyc` to:
- See logs in real time
- Monitor GPU usage
- Check the Volume status (cached models)
- Verify configured Secrets

### 10.4 Stop the backend

Modal endpoints remain active until you run:

```bash
modal app stop melodyc
```

Or from the web dashboard: **Apps** → `melodyc` → **Stop**.

> **Note on cold start:** If nobody calls the backend for more than 15 minutes (`scaledown_window=15`), Modal shuts down the container. The next call will take ~30-60 seconds to restart and reload the models.
